# PROJECT IORI CHARTER

## I. Vision（存在理由）

Project IORI (Interactive Outreach and Resource Index)は、知りたいを妨げる障壁を排除することで、誰かの「ちょっと気になる」を「生きてる理由君だらけ」に最速で変換する装置である。

本プロジェクトは、特定の人物の人気を競うためのものではない。
また、ファンの優劣や正しさを定義するためのものでもない。

人が何かに関心を持ってから愛着を持つまでの間には、多くの摩擦が存在する。
情報が散在していること、入口が分かりにくいこと、どこから触れればよいか分からないこと、理解の前に離脱してしまうこと。

それらの摩擦が取り除かれたとき、人は理解したから好きになるのではなく、
好きになる準備が整っていたことに気づく。

Project IORIは、その気づきが起きる確率を最大化するために存在する。

本プロジェクトの最終目標は閲覧者をファンに変えることではない。
誰かの人生の中に、意味を持つ対象が生まれる瞬間を増やすことである。

---

## II. Mission（継続的に果たす役割）

Project IORIは次の役割を継続的に担う。

- 迷わず到達できる入口を整理する
- 初心者が理解できない状態に陥らない構造を設計する
- 公式コンテンツへの到達を最短距離にする
- 解釈を押し付けず複数の入口を提示する
- 閲覧者が関心に沿って探索できる環境を維持する

本プロジェクトはコンテンツの代替を行わない。
価値は制作物の中に存在し、Project IORIはそれへ到達するための索引（Index）として機能する。

したがって転載・再配布・囲い込みを目的としない。
閲覧者が最終的に到達すべき場所は常に公式であり、本プロジェクトはその経路の整備に徹する。

---

## III. Mechanism（機能の仕組み）

Project IORIは以下の設計思想によって動作する。

### 1. 摩擦最小化設計
離脱の多くは興味の欠如ではなく理解コストによって発生する。
判断に時間を要しないUI、即時の再生導線、簡潔な情報配置によって認知負荷を下げる。

### 2. 複数入口モデル
人が魅力を感じる要素は一様ではない。
楽曲、パフォーマンス、声、文章、演技、人柄など複数の関心から接続できる入口を並列に配置する。

### 3. 公式還流構造
すべての導線は最終的に公式コンテンツへ接続される。
本サイトは体験の代替ではなく体験の起点として設計される。

### 4. 自律探索誘導
解釈を提示するのではなく、選択と発見の機会を増やすことで主体的関心を形成させる。

### 5. 可視化された関与
沼ゲージや選択行動により、閲覧者が自分の変化を認識できる状態をつくる。
人は自分の変化を自覚したとき、その関心を自分の意思として保持する。

---

## IV. KPI（機能しているかの指標）

Project IORIは人気や拡散量そのものを成果とみなさない。
評価は関心の深化と公式への接続によって行う。

### 到達指標
- 外部プラットフォームからの流入ユーザー数
- OGPクリック率

### 関与指標
- 滞在時間
- 動画再生クリック率
- フィルタ使用率

### 行動指標
- シェアボタン押下率
- 沼ゲージ完走率

### 還流指標（最重要）
- 公式リンククリック率

本プロジェクトにおける成功とは、サイト内に留めることではなく、
公式コンテンツへ到達させる割合が高い状態を指す。
